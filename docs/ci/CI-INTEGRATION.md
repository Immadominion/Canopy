# CI Integration Guide

This guide shows you how to integrate Canopy into your Android CI/CD pipeline using GitHub Actions.

---

## Prerequisites

1. A verified Canopy publisher account (KYC/KYB completed via the dApp Store Publisher Portal)
2. An app registered in the Canopy dashboard
3. A GitHub repository with your Solana Mobile app

---

## Setup

### 1. Get your Canopy API key

1. Open the Canopy dashboard
2. Go to **Settings → API Keys**
3. Click **Generate new key**
4. Copy the key — it is shown only once

### 2. Add secrets and variables to GitHub

In your GitHub repository:

1. Go to **Settings → Secrets and variables → Actions**
2. Under **Secrets**, add:
   - `CANOPY_API_KEY` — the API key you generated above
3. Under **Variables**, add:
   - `CANOPY_APP_ID` — the UUID of your app (Settings → App Info in the Canopy dashboard)

## GitHub Actions

### `canopy-dev/action-beta-deploy`

Uploads a signed APK to Canopy and creates a wallet-gated beta distribution track.

**Inputs**

| Name | Required | Default | Description |
|---|---|---|---|
| `api-key` | ✅ | — | Canopy API key (use a GitHub secret) |
| `app-id` | ✅ | — | App UUID from the Canopy dashboard |
| `apk-path` | ✅ | — | Workspace-relative path to the signed `.apk` |
| `version-name` | ✅ | — | Version name string (e.g. `1.2.3`) |
| `version-code` | ✅ | — | Integer version code |
| `expires-in` | ❌ | `30` | Days until the beta track expires (1–30) |
| `release-notes` | ❌ | — | Release notes shown to testers (≤ 2000 chars) |

**Outputs**

| Name | Description |
|---|---|
| `track-id` | UUID of the created beta track |
| `expires-at` | ISO 8601 expiry timestamp |
| `tester-cap` | Maximum testers allowed (always `200`) |

**Example**

```yaml
- uses: canopy-dev/action-beta-deploy@v1
  with:
    api-key: ${{ secrets.CANOPY_API_KEY }}
    app-id: ${{ vars.CANOPY_APP_ID }}
    apk-path: app/build/outputs/apk/release/app-release.apk
    version-name: "1.2.3"
    version-code: ${{ github.run_number }}
    expires-in: "14"
    release-notes: "Fix wallet connection stability on Seeker."
```

### `canopy-dev/action-release`

Runs pre-submission APK checks and creates a release record in Canopy. Use this when you're ready to submit to the dApp Store.

**Inputs**

| Name | Required | Default | Description |
|---|---|---|---|
| `api-key` | ✅ | — | Canopy API key (use a GitHub secret) |
| `app-id` | ✅ | — | App UUID from the Canopy dashboard |
| `apk-path` | ✅ | — | Workspace-relative path to the signed `.apk` |
| `version-name` | ✅ | — | Version name string |
| `version-code` | ✅ | — | Integer version code |
| `beta-track-id` | ❌ | — | UUID of the beta track that was tested (links release to beta history) |
| `release-notes` | ❌ | — | Release notes for the dApp Store submission |
| `fail-on-check-failure` | ❌ | `true` | Fail the workflow if any APK check fails |

**Outputs**

| Name | Description |
|---|---|
| `release-id` | UUID of the created release record |
| `checks-passed` | `"true"` or `"false"` |
| `check-summary` | Human-readable summary of all check results |

**APK checks performed**

| Check | Requirement |
|---|---|
| `apk_size` | ≤ 200 MB |
| `apk_extension` | File must end in `.apk` |
| `min_sdk_version` | `minSdkVersion ≥ 30` (Solana Mobile device minimum) |

> **Note:** `min_sdk_version` requires `aapt2` or `aapt` on the runner's `PATH`. This is available automatically on `ubuntu-latest` runners when `android-sdk` is set up, or after running `./gradlew assembleRelease`.

**Example**

```yaml
- uses: canopy-dev/action-release@v1
  id: release
  with:
    api-key: ${{ secrets.CANOPY_API_KEY }}
    app-id: ${{ vars.CANOPY_APP_ID }}
    apk-path: app/build/outputs/apk/release/app-release.apk
    version-name: ${{ github.ref_name }}
    version-code: ${{ github.run_number }}
    release-notes: "See CHANGELOG.md"

- name: Print release info
  run: echo "Release ID: ${{ steps.release.outputs.release-id }}"
```

## Recommended workflow pattern

```
develop branch push
  └─► canopy-dev/action-beta-deploy   (creates beta track, testers install & test)

release tag push (v*)
  └─► canopy-dev/action-release       (checks + records release)
        └─► Open Canopy dashboard → Submit to dApp Store
```

Complete example workflow files are in `docs/ci/`:

- [`example-beta-deploy.yml`](./example-beta-deploy.yml) — beta track on `develop` pushes
- [`example-release.yml`](./example-release.yml) — release record on `v*` tags

## Security notes

- Always pass `api-key` from a GitHub **secret** (`${{ secrets.CANOPY_API_KEY }}`). Never hardcode API keys in workflow files.
- Canopy API keys are hashed at rest. If a key is compromised, revoke it from the dashboard immediately.
- Signed APK download URLs generated by Canopy are wallet-bound and expire after 15 minutes. They cannot be shared across wallets.
